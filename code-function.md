# Code Skill

## Instructions

## ✅ 功能目标

- 将指定的代码逻辑**封装为独立的公共静态函数**，便于在多个模块中复用。
- 对代码进行**性能与结构上的优化**，提升其**可读性**与**执行效率**。
- **禁止修改原始变量名及业务逻辑**，确保封装过程**不影响原有功能**。

---

## 🔧 技能要求

### 1. 封装能力

- 能够识别可复用的代码段，并将其**封装为静态方法**。
- 确保函数命名符合**团队编码规范**，参数设计合理，函数职责单一。
- 封装后的函数应具备良好的**可调用性**与**可测试性**。

---

### 2. 优化能力

- 通过**重构**、**消除冗余操作**等方式提升代码质量。
- 在**不改变原有逻辑**的前提下，提高代码的**健壮性**与**可维护性**。
- 优化应聚焦于结构清晰、逻辑简洁、执行高效。

---

### 3. 约束遵守

| 约束项 | 说明 |
|--------|------|
| ❌ 不得修改变量名 | 保留原始变量名，确保与现有系统兼容 |
| ❌ 不得改变业务逻辑 | 保证封装后的行为与原始逻辑完全一致 |
| ✅ 保证可测试性 | 封装后的函数应易于单元测试与集成测试 |
| ✅ 保证可调用性 | 函数应具备清晰的输入输出接口，便于调用 |
